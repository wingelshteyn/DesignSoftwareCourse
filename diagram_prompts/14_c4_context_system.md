# Диаграмма 14. C4 Context: система ASTROLL (рисунок 14)

## Назначение
Рисунок 14 отчёта ПР8. C4 **Context** всей системы (не меняется после реструктуризации).

## Эталон (что должно получиться)
- Три персоны **сверху**: Игрок, Мастер, Администратор.
- Одна центральная система **ASTROLL** [Software System] **снизу по центру**.
- Тёмно-синие блоки C4, белый текст.
- Стрелки с **развёрнутыми русскими подписями** (как MDT рис. 14).

## Промпт для генерации
```
Нарисуй C4 System Context Diagram для ASTROLL (рис. 14 MDT).

Layout: три Person сверху в ряд, System ASTROLL снизу по центру, тёмно-синий стиль C4.

Persons:
- Игрок — хранит персонажей, подключается к комнатам, делает броски, редактирует доску
- Мастер — создаёт комнаты, управляет участниками, ведёт сцену, сохраняет доски
- Администратор — управляет пользователями, настройками, аудитом

System:
- ASTROLL [Software System] — веб-платформа для проведения онлайн настольных RPG: комнаты, VTT-доска, персонажи, броски, друзья

Подписи на стрелках (русский):
- Игрок → ASTROLL: хранит персонажей, играет в комнате, делает броски
- Мастер → ASTROLL: ведёт сессию, управляет участниками, сохраняет доски
- Администратор → ASTROLL: управляет пользователями и эксплуатационными параметрами
```

## PlantUML (готовая реализация)
```plantuml
@startuml
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Context.puml

LAYOUT_TOP_DOWN()

Person(player, "Игрок", "Игрок настольных RPG,\nхранит персонажей")
Person(master, "Мастер", "Ведущий сессии,\nсоздатель комнаты")
Person(admin, "Администратор", "Эксплуатация системы,\nуправление пользователями")

System(astroll, "ASTROLL", "Веб-платформа для онлайн\nнастольных RPG: комнаты,\nVTT-доска, персонажи, броски")

Rel(player, astroll, "Хранит персонажей,\nиграет в комнате,\nделает броски")
Rel(master, astroll, "Ведёт сессию,\nуправляет участниками,\nсохраняет доски")
Rel(admin, astroll, "Управляет пользователями\nи эксплуатационными\nпараметрами")
@enduml
```
