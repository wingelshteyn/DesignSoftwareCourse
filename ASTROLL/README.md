# ASTROLL

Демонстрационная сервисная архитектура ASTROLL после реструктуризации.

Система разделена на крупные сервисы, соответствующие ограниченным контекстам DDD. Общие DTO и события вынесены в `shared-contracts`, чтобы сервисы не зависели от внутренних моделей друг друга.

## Сервисы

- `auth_accounts` - пользователи, профиль, JWT, роли.
- `social` - друзья и поиск пользователей.
- `characters` - персонажи, листы, портреты, импорт и экспорт.
- `session` - комнаты, права, команды игровой сессии.
- `board` - элементы доски и снапшоты.
- `dice` - броски кубов и история.
- `administration` - настройки, аудит и эксплуатационные политики.
- `realtime_worker` - публикация realtime-событий и фоновое сохранение снапшотов.

## Инфраструктура

API Gateway маршрутизирует HTTP и WebSocket-трафик. Redis используется как cache/broker, PostgreSQL - как транзакционное хранилище, Object Storage - для карт, портретов, аватаров и снапшотов.
