# ASTROLL

Демонстрационная сервисная архитектура ASTROLL после реструктуризации.

Система разделена на крупные сервисы, соответствующие ограниченным контекстам DDD. Общие DTO и события вынесены в `shared-contracts`, чтобы сервисы не зависели от внутренних моделей друг друга.

## Целевые сервисы

- `identity` - пользователи, профиль, JWT, роли, друзья и поиск пользователей.
- `characters` - персонажи, листы, портреты, импорт и экспорт.
- `session` - комнаты, права, команды игровой сессии, публикация realtime-событий и WebSocket-рассылка через внутренний worker.
- `board` - элементы доски и снапшоты.
- `dice` - броски кубов и история.
- `administration` - настройки, аудит и эксплуатационные политики.

## Инфраструктура

API Gateway маршрутизирует HTTP и WebSocket-трафик. Redis используется как cache/broker для `session`, PostgreSQL - как транзакционное хранилище, Object Storage - для карт, портретов, аватаров и снапшотов.

Текущий учебный скелет может сохранять старые папки `auth_accounts`, `social` и `realtime_worker`; в целевой архитектуре их ответственность объединяется в `identity` и `session`.
