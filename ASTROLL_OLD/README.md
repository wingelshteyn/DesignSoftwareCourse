# ASTROLL_OLD

Демонстрационная реализация ASTROLL в виде модульного монолита.

Структура повторяет учебный подход `MetalDefectTracker_old`: один backend-процесс, общая база данных и набор Django-приложений, разделенных по доменным границам.

## Модули

- `accounts` - учетные записи, профиль, роли и JWT.
- `friends` - поиск пользователей и список друзей.
- `characters` - персонажи, JSON-листы, импорт и экспорт.
- `rooms` - комнаты, roster, роль мастера и права участника.
- `board` - элементы доски, снапшоты, адаптер Excalidraw.
- `dice` - броски кубов и история.
- `game_session` - смысловое ядро: команды сессии, права, realtime-события.
- `administration` - настройки, аудит и эксплуатационные политики.

Код носит архитектурный характер: он фиксирует связи, паттерны и границы модулей, а не реализует полный production backend.
